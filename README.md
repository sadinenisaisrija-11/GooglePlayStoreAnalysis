# 📱 Google Play Store Analysis

## 📌 Project Overview

This project performs Exploratory Data Analysis (EDA) on the Google Play Store dataset to understand app trends, user ratings, installs, pricing, and customer sentiment. The analysis includes data cleaning, visualization, pricing analysis, and sentiment analysis of user reviews to provide meaningful business insights for app developers.

---

## 🎯 Objective

- Load and clean Google Play Store datasets.
- Analyze app categories and their distribution.
- Explore app ratings and install trends.
- Compare free and paid applications.
- Estimate revenue generated by paid apps.
- Perform sentiment analysis on user reviews using TextBlob.
- Create interactive visualizations using Plotly.
- Generate insights to help developers launch successful applications.

---

## 📂 Dataset

This project uses the following datasets from Kaggle:

- **googleplaystore.csv** – App information
- **googleplaystore_user_reviews.csv** – User reviews

**Dataset Source:**
https://www.kaggle.com/datasets/lava18/google-play-store-apps

## 🧹 Data Cleaning

The following preprocessing steps were performed:

- Removed duplicate records
- Handled missing values
- Fixed incorrect data types
- Converted **Installs** column to numeric values
- Converted **Price** column to float
- Removed corrupted records
- Prepared clean datasets for analysis

# 📈 Key Insights

- Family and Game categories contain the highest number of applications, making them the most competitive categories.
- Most applications available on the Google Play Store are free.
- Applications with higher ratings generally receive more installs.
- Positive user reviews dominate across most categories.
- Paid apps with large install counts generate higher estimated revenue.
